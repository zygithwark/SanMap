# Contributing to SanMap

Thank you for your interest in contributing to SanMap! This document outlines the process for contributing to the project, reporting bugs, and suggesting new features.

## How to Contribute

### 1. Reporting Bugs
If you find a bug, please open an issue in the GitHub repository with the following information:
- **Clear title** describing the bug
- **Steps to reproduce** the bug
- **Expected behavior**
- **Actual behavior**
- **Screenshots** (if applicable)
- **Device/OS version** (if relevant)

### 2. Suggesting New Features
We welcome feature suggestions! Open an issue with:
- **Feature title**
- **Detailed description** of the feature and its use case
- **Why it would be valuable** for the project
- **Any mockups or references** (if available)

### 3. Code Contributions
If you'd like to contribute code:
1. **Fork the repository** to your own GitHub account.
2. **Create a new branch** for your changes (`git checkout -b feature/your-feature-name`).
3. **Make your changes** and commit them with clear, descriptive commit messages.
4. **Push your branch** to your fork.
5. **Open a pull request** (PR) to the `main` branch of the original repository.

### 4. Pull Request Guidelines
- Keep PRs focused on a single feature or bug fix.
- Include a clear description of what the PR does and why.
- Ensure your code follows the existing style (we'll establish a style guide as the project matures).
- Update documentation if needed.
- Make sure all tests pass (once tests are added).

### 5. Code of Conduct
We aim to foster an inclusive and respectful community. Please be kind and considerate in all interactions.

## Getting Help
If you have questions or need help, feel free to comment on relevant issues or open a new issue with the "question" label.

Thank you for helping us improve sanitation data collection worldwide!
